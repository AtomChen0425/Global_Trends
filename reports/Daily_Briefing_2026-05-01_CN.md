**背景**

本文讨论了 Linux 内核中的一个本地权限提升漏洞 CVE-2026-31431，代号为 "CopyFail"。该漏洞的根源在于内核版本 4.14 中引入的一个错误（commit 72548b093ee38a6d4f2a19e6ef1948ae05c181f7）。此漏洞被描述为“近年来最严重的内核‘提权’漏洞之一”。

**技术实现与修复**

漏洞的修复工作在多个内核版本中进行，例如 6.18.22（commit fafe0fa2995a0f7073c1c358d7d3145bcc9aedd8）、6.19.12（commit ce42ee423e58dffa5ec03524054c9d8bfd4f6237）和 7.0（commit a664bf3d603dc3bdcf9ae47cc21e0daec706d7a5）。然而，将此修复回溯到更早的长期支持版本（如 6.12, 6.6, 6.1, 5.15, 5.10）面临挑战，可能由于 API 变更导致回溯不直接。文章中提供了一个针对该漏洞的临时规避补丁（0001-crypto-disable-authencesn-module-for-CVE-2026-31431.patch），该补丁通过禁用 `authencesn` 模块来缓解问题，尽管作者承认这可能不是最优解。

**应用场景与影响**

该漏洞影响广泛，因为其引入时间较早（2017年），且未及时回溯到所有长期支持的内核版本，意味着大量运行旧版本内核的 Linux 服务器可能面临风险。文章还提及了 Linux 发行版在处理此类漏洞时的信息披露流程，指出除非报告者主动通知 `linux-distros` 邮件列表，否则发行版可能不会提前收到通知。

**总结**

CVE-2026-31431 是一个严重的 Linux 内核本地权限提升漏洞，其修复工作已在较新内核版本中完成。然而，由于回溯修复的复杂性，较旧的内核版本仍可能受此漏洞影响。此事件也凸显了内核漏洞披露流程以及发行版及时获取安全信息的重要性。建议用户及时更新内核版本，或根据实际情况评估和应用已提供的临时规避措施。

### 5. [Representation Fréchet Loss for Visual Generation](https://arxiv.org/abs/2604.28190v1)

**背景**

Fréchet Distance (FD) 作为生成模型训练目标长期以来被认为难以有效优化。本文提出了一种名为 FD-loss 的新方法，旨在解决这一挑战。核心思想是将用于 FD 估计的总体样本数量（例如 50,000）与用于梯度计算的批次大小（例如 1024）解耦，从而在表示空间中实现 FD 的有效优化。

**技术实现与应用场景**

FD-loss 的应用带来了多项令人惊喜的发现。首先，在不同的表示空间中，使用 FD-loss 对基础生成器进行后训练，能够显著提升生成样本的视觉质量。例如，在 Inception 特征空间中，经过优化的一步生成器在 ImageNet 256x256 数据集上取得了 0.72 的 FID 分数。其次，FD-loss 能够将多步生成器有效地转化为强大的单步生成器，且无需借助教师蒸馏、对抗训练或逐样本目标等技术。此外，研究还揭示了 FID 指标可能存在的局限性：一些现代表示空间虽然在 Inception FID 上得分更高，但生成的样本质量却不一定更优。这促使了 FDr$^k$ 的提出，一个结合了多重表示空间的评估指标。

**总结**

本文提出的 FD-loss 方法为在表示空间中优化 Fréchet Distance 提供了一种可行且有效的途径。该方法不仅能够提升生成模型的视觉质量，还能实现生成器的重用和优化。同时，研究也指出了现有评估指标的不足，并提出了 FDr$^k$ 作为一种更全面的评估方案。这项工作有望鼓励研究人员在生成模型的训练目标和评估指标方面，进一步探索在多样化表示空间中使用分布距离的潜力。
